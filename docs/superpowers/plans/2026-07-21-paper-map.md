# Paper Map Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an accessible Paper Map tab that deterministically organizes manifest sections, assets, and locally detected citations.

**Architecture:** Extend the cached browser analysis pass with per-page citations using the existing detector, then feed a pure tree builder that owns hierarchy and assignment. A focused client component renders the resulting tree; `ExploreShell` only owns tab selection.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript 5, pdf.js 6, Tailwind CSS 4, Vitest 4.

## Global Constraints

- Do not add `mentions[]` or citation occurrences to the server manifest.
- Coordinates remain normalized top-left and are never converted in this feature.
- Render no dead affordance: unresolved references are text, not links.
- Reuse `sectionForPage`; do not create a competing page-to-section rule.
- Do not modify `Reader.tsx`, extraction code, or generated schema types.
- Add no dependency and make no LLM call.
- Preserve all existing reader behavior and tests.

### Task 2: Build the pure Paper Map model

**Files:**
- Create: `apps/web/lib/explore/paper-map.ts`
- Create: `apps/web/lib/explore/paper-map.test.ts`

**Interfaces:**
- Consumes: `Manifest`, `Mention[][]`, `Citation[][]`, and `sectionForPage(manifest, page)`.
- Produces: `buildPaperMap(manifest, mentionsByPage, citationsByPage): PaperMapModel`.

The public model is:

```ts
export interface PaperMapAsset {
  asset: Asset;
  mentionPages: number[];
}

export interface PaperMapCitation {
  reference: Reference;
  page: number;
  surface: string;
}

export interface PaperMapSection {
  id: string;
  title: string;
  page: number | null;
  level: number;
  assets: PaperMapAsset[];
  citations: PaperMapCitation[];
  children: PaperMapSection[];
}

export interface PaperMapModel {
  title: string;
  sections: PaperMapSection[];
  objectCount: number;
}
```

- [ ] **Step 1: Write failing hierarchy and assignment tests**

Create a minimal manifest with level-1 and level-2 sections, assets before and after the
first heading, and citations on several pages. Assert:

```ts
const map = buildPaperMap(manifest, mentionsByPage, citationsByPage);
expect(map.sections.map((section) => section.title)).toEqual(["Unsectioned", "1 Intro", "2 Method"]);
expect(map.sections[2].children.map((section) => section.title)).toEqual(["2.1 Details"]);
expect(map.sections[2].assets.map(({ asset }) => asset.asset_id)).toEqual(["fig-1"]);
expect(map.sections[2].children[0].assets.map(({ asset }) => asset.asset_id)).toEqual(["tab-1"]);
```

Add separate tests proving that citation occurrences are deduplicated per section by
`ref_id`, unresolved references are retained with `openable: false`, empty real sections
remain present, and all manifest assets assigned exactly once.

- [ ] **Step 2: Run the test and verify RED**

Run: `npm test -- lib/explore/paper-map.test.ts`

Expected: FAIL with module-not-found for `./paper-map`.

- [ ] **Step 3: Implement the minimal tree builder**

Build section nodes in manifest order using a stack. Pop while the stack top has
`level >= next.level`, attach the next node to the remaining parent or the root, and retain
flat `id -> node` lookup. Assign page-bound objects using `sectionForPage`; map its
`sectionId` back to the node. Lazily create one root `Unsectioned` node only when something
cannot be assigned.

For each asset, derive distinct sorted mention pages from `mentionsByPage`. For each
citation occurrence, resolve `refIds` against `manifest.references`, keep one occurrence
per `(section id, ref id)`, and preserve the first source page/surface.
